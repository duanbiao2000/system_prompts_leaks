## 🎴 闪卡练习 | Flashcard Exercises

### 第1组: 程度修饰语 | Degree Modifiers

> [!question] Q1: "appropriately cite" vs "cite correctly"?
>

> <summary>查看答案</summary>
>
> **appropriately** = 正确性 + 语境适配 + 风格一致<br>
> **correctly** = 仅格式正确性
>
> **关键差异**: appropriately 涵盖适配性和风格，correctly 仅关注准确性


> [!question] Q2: strategically vs carefully?
>
> <summary>查看答案</summary>
>
> **strategically** = 策略决策，涉及权衡和目标考量<br>
> **carefully** = 仔细执行，关注准确性和细节
>
> **使用场景**:
> - strategically: "decide strategically how many searches to run"
> - carefully: "carefully analyze the search results"


> [!question] Q3: thoroughly vs comprehensively?
>
> <summary>查看答案</summary>
>
> **thoroughly** = 强调过程的完整性（不遗漏）<br>
> **comprehensively** = 强调范围的广泛性（覆盖多方面）
>
> **组合使用**: "thoroughly and comprehensively" = 既完整又全面


> [!question] Q4: naturally vs typically?
>
> <summary>查看答案</summary>
>
> **naturally** = 符合本性，预期的<br>
> **typically** = 大多数情况，统计性的
>
> **微妙差异**: naturally 更像"应该这样"，typically 更像"通常这样"


### 第2组: 语气控制词 | Tone Controllers

> [!question] Q5: MUST vs strongly prefer?
>
> <summary>查看答案</summary>
>
> **MUST** = 100% 强制，无例外<br>
> **strongly prefer** = 90% 建议，极少例外
>
> **示例**:
> - MUST: "MUST cite sources" → 不引用 = 违规
> - strongly: "strongly consider thinking" → 不思考可能也可以


> [!question] Q6: CRITICAL vs IMPORTANT?
>
> <summary>查看答案</summary>
>
> **CRITICAL** = 必须遵守，违规严重<br>
> **IMPORTANT** = 应该遵守，违规影响质量
>
> **记忆口诀**:
> - CRITICAL → 生存问题
> - IMPORTANT → 质量问题


> [!question] Q7: NEVER vs do not?
>
> <summary>查看答案</summary>
>
> **NEVER** = 绝对禁止，零容忍<br>
> **do not** = 强烈禁止，极少例外
>
> **记忆提示**: NEVER = 大写强调 = 零容忍


> [!question] Q8: should vs may?
>
> <summary>查看答案</summary>
>
> **should** = 建议遵循，默认应该<br>
> **may** = 可以选择，允许但不强制
>
> **强制程度**: should (70%) > may (30%)


### 第3组: 边界词 | Boundary Words

> [!question] Q9: unless vs except if?
>
> <summary>查看答案</summary>
>
> **unless** = "如果不"（单一条件）<br>
> **except if** = "除了如果"（明确排除）
>
> **示例**:
> - unless asked: 如果不问就不做
> - except if asked: 除了问的情况都不做


> [!question] Q10: at most vs maximum?
>
> <summary>查看答案</summary>
>
> **at most** = 软性上限，建议性<br>
> **maximum** = 硬性上限，强制性
>
> **使用场景**:
> - at most: 最佳实践、资源优化
> - maximum: 安全限制、硬约束


> [!question] Q11: within vs under?
>
> <summary>查看答案</summary>
>
> **within** = ≤ (小于等于，包括边界)<br>
> **under** = < (严格小于，不包括边界)
>
> **示例**:
> - within 15: 15词可以
> - under 15: 最多14词


> [!question] Q12: approximately vs exactly?
>
> <summary>查看答案</summary>
>
> **approximately** = 大约，允许±20%浮动<br>
> **exactly** = 精确，不能偏离
>
> **系统提示词选择**: approximately 更灵活，exact 仅在必要时使用


### 第4组: 微妙表达 | Nuanced Expressions

> [!question] Q13: "breaks continuity" 的隐含意义？
>
> <summary>查看答案</summary>
>
> **答案**: 暗示"必须避免"的负面后果
>
> **设计哲学**: 描述后果而非下达命令更有效
> - 直接命令: "You must detect these cues"
> - 暗示后果: "Missing cues breaks continuity"
>
> **效果**: AI 理解"为什么"，主动避免


> [!question] Q14: politely 的作用？
>
> <summary>查看答案</summary>
>
> **答案**: 礼貌地传达不受欢迎的内容
>
> **关键**: politely 修饰"怎么说"，不是"说什么"
>
> **示例**:
> - ❌ 过度委婉: "I'm afraid I couldn't find..."
> - ✅ 礼貌但明确: "The search results do not contain this information"


> [!question] Q15: substantive vs generic?
>
> <summary>查看答案</summary>
>
> **substantive** = 有实质内容的<br>
> **generic** = 泛泛的，缺乏特异性
>
> **示例**:
> - Substantive: "robot", "OAuth", "machine learning"
> - Generic: "discuss", "thing", "problem"


> [!question] Q16: forces vs makes?
>
> <summary>查看答案</summary>
>
> **forces** = 被迫、不合理、负面<br>
> **makes** = 导致、中性、描述性
>
> **使用**: forces 用于暗示必须避免的行为


### 第5组: 操作指导词 | Action Guides

> [!question] Q17: Extract vs Get?
>
> <summary>查看答案</summary>
>
> **Extract** = 识别并筛选后获取<br>
> **Get** = 直接获取，无筛选
>
> **系统提示词选择**: Extract 更精确


> [!question] Q18: Avoid vs Don't?
>
> <summary>查看答案</summary>
>
> **Avoid** = 主动识别并绕开低质量<br>
> **Don't** = 简单禁止某行为
>
> **差异**: Avoid 暗示有更好的选择


> [!question] Q19: Synthesize vs Combine?
>
> <summary>查看答案</summary>
>
> **Synthesize** = 认知整合，理解后创造统一输出<br>
> **Combine** = 物理组合，简单拼接
>
> **质量**: Synthesize 产生更连贯的输出


> [!question] Q20: Acknowledge vs Mention?
>
> <summary>查看答案</summary>
>
> **Acknowledge** = 承认并透明化来源<br>
> **Mention** = 简单提及某事
>
> **价值**: Acknowledge 建立信任

### 练习 5: 设计完整指令 | Design Complete Instructions

> [!question] 为"AI 编程助手"设计包含五类词汇的系统指令

<details>
<summary>查看答案</summary>

**答案**:

```markdown
## AI 编程助手系统指令

### CRITICAL 规则
CRITICAL: NEVER execute code without user confirmation
CRITICAL: MUST warn about security vulnerabilities

### MUST 规则
MUST ALWAYS explain code changes
MUST provide accurate syntax examples

### NEVER 规则
NEVER claim to test code without actually running it
NEVER hide errors from the user

### 强烈建议
strongly prefer to include error handling
strongly consider suggesting improvements

### 一般指导
should be concise for simple questions
should provide details for complex problems

### 可选行为
may offer to run code in sandbox
may ask clarifying questions about requirements

### 操作流程
1. **Carefully** analyze the user's request
2. **Extract** the core requirements
3. **Strategically** decide the best approach
4. **Synthesize** a solution naturally
5. **Politely** explain the code
6. **Acknowledge** any assumptions made
```

**使用的词汇类别**:
- CRITICAL/NEVER/MUST: 语气控制词
- strongly/should/may: 语气控制词
- carefully/strategically/naturally: 程度修饰语
- politely: 程度修饰语/微妙表达
- Extract/Synthesize/Acknowledge: 操作指导词
</details>

### 练习 6: 词汇组合设计 | Vocabulary Combination Design

> [!question] 使用词汇组合表达以下要求

"要求 AI 在不确定时主动提议搜索，但不要过度搜索"

<details>
<summary>查看答案</summary>

**答案**:

```markdown
"Consider whether the user's question involves information that may have changed.
When uncertain, **offer** to **strategically** search for current information.
**Avoid** excessive tool calls - use the **minimum** number **necessary**,
**approximately** 1-3 searches for simple queries, **up to** 5 for complex topics.
**Unless** the user **explicitly** requests comprehensive research,
**prioritize** efficiency over thoroughness."
```

**词汇组合分析**:
- **Consider**: 评估决策（操作指导）
- **offer**: 主动提议（操作指导）
- **strategically**: 策略性（程度修饰）
- **Avoid**: 主动避免（操作指导）
- **minimum/necessary**: 数量下限（边界词）
- **approximately**: 近似值（边界词）
- **up to**: 软性上限（边界词）
- **Unless**: 条件例外（边界词）
- **explicitly**: 明确性（程度修饰）
- **prioritize**: 排序优先（操作指导）
</details>

### 练习 7: 版本演进预测 | Version Evolution Prediction

> [!question] 基于 Claude 3.7 → 4.1 → 4.5 的演进，预测 Claude 5.0 的新词汇特征

<details>
<summary>查看答案</summary>

**预测答案**:

**Claude 5.0 可能的新词汇特征**:

1. **主动性词汇增强**
   - 新词: "proactively", "anticipate", "preempt"
   - 理由: 从响应式到主动式

2. **透明度词汇增强**
   - 新词: "explicitly disclose", "transparently", "reveal reasoning"
   - 理由: 用户对 AI 决策过程要求更高透明度

3. **体验优化词汇**
   - 新词: "gracefully", "tactfully", "thoughtfully", "seamlessly"
   - 理由: 从功能正确到体验优化

4. **协作性词汇**
   - 新词: "collaboratively", "jointly", "in partnership"
   - 理由: AI 与用户的协作关系深化

**演进方向**: 从"控制行为"到"优化体验"再到"深度协作"
</details>
